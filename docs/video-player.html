VideoPlayer

The main component to play video with default configs.

Usage

import VideoPlayer from "react-native-video-controller";
render(){
  return(
   <VideoPlayer
     source={{ uri: source }} // Can be a URL or a local file.
     ref={ref => {
       this.videoplayer= ref;
     }} // Store reference
     style={[styles.videoContainerStyle, { overflow: "hidden" }]}
     videoStyle={styles.videoStyle}
     // style={styles.backgroundVideo}
     resizeMode="contain"
     // controls={fullScreen}
     bottom={this.bottom} />
  )
}

Note: the prop bottom and renderTop* should be a function that gets the state of the component as argument and returns a react component

type State = {
source: { uri: string },
controls_hidden: boolean,
currentTime: number,
seekableDuration: number,
show_video: boolean,
playState: PLAYING | PAUSED | LOADING | ERROR,
resolutions: Array<number>,
paused: boolean,
fullscreen: boolean,
subtitles: Array<any>,
currentSub: string,
selectedVideoTrackHeight: number
};

where playing state is a constant string that can be imported like:

import { PlayState } from "react-native-video-controller";
...
if( this.state.playState === PlayState.PAUSED){
 console.log('the video is paused! ')
}

To access the underlying player's methods use player attribute, ie:

this.videoplayer.player.seek(12)

Props

source (required)
Type: any

video source to play, will be passed to react-native-video

renderPlayPause
Type: (typeof Animated.Value) => React.Node
Default value: (progress: typeof Animated.Value) => ( <PlayPause progress={progress} /> )

render a custom play/pause component. the argument is an AnimatedValue that will be animated from 0 to 0.5 when translating from PAUSE to PLAY, and from 0.5 to 1 when translating from PLAY to PAUSE. this way is useful if you want to animate different parts of a lottie animation for play/pause. also you can interpolate the value to get a monotonic value from PLAY to PAUSE.

renderTopRight (required)
Type: State => React.Node

render component for the top right

renderTopLeft (required)
Type: State => React.Node

render component for the top left

renderLoading
Type: () => React.Node
Default value: () => <Loading />

render component for loading

renderError (required)
Type: () => React.Node

render component for error

bottom (required)
Type: State => React.Node

Methods

These methods can be accessed on the ref of the component, e.g. videoPlayerRef.setResolution(...args).

setResolution
Params: height: number

changes the current resolution of the video

setSub
Params: subtitle

changes the current subtitle

play

play the video

pause

pause the video

togglePlayback

toggle palyback state

reload

reload the video (for example in case of an error) this will actually rerender the component.

toggleFullScreen

toggles fullscreen mode. in full screen mode the bottom prop will be rendered over the player, and the player will positioned absolutely.

toggleControls

toggle controls on the screen

hide_controls_with_timeout

hide controls after 5s delay

Static properties

These properties can be accessed on VideoPlayer by using the dot notation, e.g. VideoPlayer.secondToTime.

secondToTime
Type: Function
Params: seconds: number
Returns: string

converts second to human readable format as hh:mm:ss
